FILE I/O RELATED FEATURES

All the file input/output related features of Flint are contained within the FILE main menu. These are;

  • READ-CASE
  • WRITE-CASE
  • WRITE-GRID
  • READ-DATA
  • WRITE-DATA
  • DOS-SHELL
  • QUIT
READ-CASE command reads Geometry, Grid, Cell and Boundary information which had been stored into a file by means of the 'WRITE-CASE' command during a previous Flint session. These case files are stored unformatted in a computer dependent way. Therefore they can not be read or manipulated by any other means except 'Flint'.

By default all case files are stored using the .CAS filename extension, so that they can be easily identified. File read menu normally displays only the filenames with the .CAS extension as well as any subdirectories and also the parent directory as ... This will enable the Flint user to change directories and search through them by simply clicking on them.
As well as all the files matching the search pattern, which is *.CAS by default and the local directories, there are also the following three entries on the user selection list these are;

  • QUIT: Quit file read operation to return to Flint menus.
  • CHANGE-DRIVE: Search other hard,network or floppy drives.
  • WILD-SELECT: Change the default file search pattern (from *.CAS ).
WRITE-CASE command will save all the current Geometry, Grid, Cell and Boundary information into a file. The user will normally be offered a default name which can be modified by the user. The default name will always have the .CAS filename extension. It is strongly recommended that this extension is used for all saved case files.
WRITE-GRID command will save the current Grid and Cell information as a Formatted PREBFC file. These files can later be read into all versions higher then V4.11 of Fluent. It is also possible to convert these 2D grids into 3D by using Fluents 3D-from-2D command. Restricted versions of Flint/2D have this facility disabled.
READ-DATA command reads all the field information which had been stored into a file by means of the 'WRITE-DATA' command during a previous Flint session. These data files are written unformatted and therefore can not be read or manipulated by any other means except 'Flint'.

By default all data files are stored using the .DAT filename extension, so that they can be easily identified. File read menu normally displays only the filenames with the .DAT extension as well as any subdirectories and also the parent directory as ... This will enable the Flint user to change directories and search through them by simply clicking on them.
As well as all the files matching the search pattern, which is *.DAT by default and the local directories, there are also the following three entries on the user selection list these are;

  • QUIT: Quit file read operation to return to Flint menus.
  • CHANGE-DRIVE: Search other hard,network or floppy drives.
  • WILD-SELECT: Change the default file search pattern (from *.DAT ).
WRITE-DATA command will save the current solution information i.e. the complete set of field variables for all the grid-locations into an unformatted data file. The default filename extension is .DAT. It is strongly recommended that this extension is used for all saved data files.
DOS-SHELL command starts up a command shell in which DOS commands can be executed. This is only useful for the DOS version of FLINT where multi-tasking is not possible. User can return back to FLINT by typing EXIT in this command window.
QUIT command simply terminates a FLINT session. It is important to save your geometry, grid and field data information by using the WRITE-CASE and WRITE-DATA commands if you have made any modifications and you wish to refer to them during future Flint sessions.
