DEFINE PROBLEM

When a problem is defined, the default physical model used by Flint is that of laminar flow with no heat transfer. However, SETUP -> DEFINE-PROBLEM command will allow the user to switch on extra features to solve for. These features are:

  • Turbulant-Flow ( k-Epsilon model only)
  • Heat-transfer calculations.
  • W-Velocities calculations(????)
By default Flint solves the momentum and mass conservation equations in terms of u,v (and w for 3D ) velocity components and pressures (p).

Switching on the heat-transfer model introduces the energy conservation equation which is solved in terms of enthalpy values.

Switching on the turbulence model introduces two extra equations which are solved in terms of Turbulence Energy (E) and Turbulence Dissipation (D), by using the two-equation k-epsilon turbulance model.

These features will in turn have a direct bearing on the types of boundary conditions that will need to be supplied. For example the walls will now need to have predefined temperatures or heat-flux values and turbulence quantities must be specified at the inlet boundaries.
